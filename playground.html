Lit Reactivity System

Lit uses decorators to create reactive properties that automatically trigger re-renders when changed.

@property External Properties

Exposed to users via HTML attributes. Changes trigger the update cycle.

@property({ type: String }) text = ''; @property({ type: Boolean, attribute: 'is-streaming' }) isStreaming = false; @property({ type: Number, attribute: 'throttle-ms' }) throttleMs = 50;

@state Internal State

Private reactive state. Not exposed as attributes but still triggers updates.

@state() private _throttledText = ''; @state() private _hasStreamed = false; // Non-reactive (no decorator) private _adaptiveThrottleMs = 0; private _lastMorphDuration = 0;

Lit Lifecycle Methods

The update cycle flows through several lifecycle methods, each serving a specific purpose.

connectedCallback()

Element added to DOM. Set up event listeners, make focusable for cursor styling.

override connectedCallback(): void { super.connectedCallback(); this.addEventListener('click', this._handleClick); // Make component focusable for cursor styling if (!this.hasAttribute('tabindex')) { this.setAttribute('tabindex', '0'); } }
willUpdate(changedProperties)

Called before render. Latch streaming state, ensure KaTeX loaded, handle throttling.

override willUpdate(changedProperties: PropertyValues): void { if (changedProperties.has('isStreaming')) { if (this.isStreaming) { this._hasStreamed = true; this._resetStreamingStats(); this.focus(); // Auto-focus when streaming starts } else { this._logStreamingStats(); // Log stats when done } } if (changedProperties.has('text') && this.text) { this._ensureKaTeXLoaded(); // Lazy load math } }
render()

Return the template. Should be pure (no side effects).

override render() { if (this._useSyncStrategy) { // Empty container for morphing return html`<div class="markdown"></div>`; } else { // Direct render for static content return html`<div class="markdown">\${unsafeHTML(this._renderedContent)}</div>`; } }
updated(changedProperties)

Called after render. Apply optimized morphing, track stats, adjust throttle.

override updated(changedProperties: PropertyValues): void { if (this._useSyncStrategy && this._containerRef) { const startTime = performance.now(); morphContentOptimized(this._containerRef, this._renderedContent); this._lastMorphDuration = performance.now() - startTime; this._adjustAdaptiveThrottle(); // Dynamic throttle adjustment this._trackMorphStats(); } }

ðŸ†• Optimized Element-Level Morphing

Instead of morphing the entire container, we now hash and compare each top-level element, skipping unchanged nodes entirely.

Performance Win: During streaming, only new/changed paragraphs are morphed. Existing content is untouched, dramatically reducing DOM operations.

export function morphContentOptimized(container: Element, newHtml: string): boolean { const temp = document.createElement('div'); temp.innerHTML = newHtml; for (let i = 0; i < maxLen; i++) { const newHash = simpleHash(newChild.outerHTML); if (prevElementHashes[i] !== newHash) { // Element changed - morph it Idiomorph.morph(oldChild, newChild.outerHTML, { morphStyle: 'outerHTML' }); stats.updated++; } else { // Element unchanged - skip! stats.skipped++; } } prevElementHashes = newHashes; return stats.updated > 0; }

private _adjustAdaptiveThrottle(): void { const morphTime = this._lastMorphDuration; const targetMorphBudget = 0.25; // 25% of interval // Calculate ideal throttle based on last morph time const idealThrottle = morphTime / targetMorphBudget; // Smooth adjustment (30% toward ideal each update) const smoothingFactor = 0.3; const newThrottle = currentThrottle + (idealThrottle - currentThrottle) * smoothingFactor; // Clamp: never below base, max 4x base or 200ms this._adaptiveThrottleMs = Math.max(baseThrottle, Math.min(maxThrottle, newThrottle)); }
